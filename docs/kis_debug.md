# KIS Debug Tool (kis_debug.py)

## 개요
한국투자증권(KIS) Open API의 기능을 종합적으로 테스트하고 디버깅하는 도구입니다. 
주식 거래 실패 문제나 API 연결 문제를 진단할 때 사용합니다.

## 파일 위치
`Scripts/tests/kis_debug.py`

## 주요 기능

### 1. 기본 KIS API 진단
- KIS 인증 테스트
- API 연결 상태 확인
- 계좌 정보 검증
- 기본 API 호출 테스트

### 2. 수정된 기능 테스트 (2026-02-18 추가)
- **계정 잔고 조회 테스트**: OPSQ2000 오류 수정 검증
- **주식 가격 조회 테스트**: 삼성전자 현재가 조회
- **채권 가격 조회 테스트**: KR103502GA34 채권 가격 조회
- **보유종목 조회 테스트**: 현재 보유 주식/채권 목록

### 3. 거래 모듈 테스트
- 매수 주문 테스트 (낮은 가격으로 비체결 주문)
- 주문 실행 검증

### 4. 웹 서버 테스트 (2026-02-22 추가)
- 포트폴리오 웹 서버 시작/종료 테스트
- API 엔드포인트 동작 확인
- 포트폴리오 데이터 조회 검증

### 5. 데이터베이스 테스트 (2026-02-22 신규 추가)  
- **DB 연결 테스트**: PostgreSQL 연결 상태 확인
- **테이블 데이터 확인**: 각 테이블별 저장된 데이터 개수 및 샘플 출력
- **데이터 저장 테스트**: 샘플 데이터 생성 및 저장 검증
- **BitCoin_Auto 스타일 연결**: app.py 방식을 참고한 안정적 DB 연결

## 테스트 항목 상세

### 1. 계정 잔고 조회 테스트
```python
balance = portfolio_fetcher.fetch_account_balance()
```
- **목적**: OPSQ2000 오류 수정 확인
- **성공 시**: 현금 잔고 정보 출력
- **실패 시**: API 파라미터 형식 문제

### 2. 주식 가격 조회 테스트
```python
stock_price = portfolio_fetcher.fetch_current_price("005930")
```
- **대상**: 삼성전자 (005930)
- **목적**: 주식 가격 조회 API 정상 작동 확인
- **성공 시**: 실시간 주가 출력

### 3. 채권 가격 조회 테스트
```python
bond_price = portfolio_fetcher.fetch_current_price("KR103502GA34")
```
- **대상**: 국고채권 (KR103502GA34)
- **목적**: 새로 구현된 채권 가격 조회 기능 확인
- **주의**: 장 시간이나 종목에 따라 가격이 없을 수 있음

### 4. 보유종목 조회 테스트
```python
holdings = portfolio_fetcher.fetch_holdings()
```
- **목적**: 현재 계좌의 보유 주식/채권 확인
- **성공 시**: 보유종목 딕셔너리 출력

## 문제 해결 가이드

### OPSQ2000 오류가 계속 발생하는 경우
1. 계정 번호 형식 확인 (8자리)
2. 상품 코드 확인 (01)
3. 파라미터 대소문자 확인

### 채권 가격이 0으로 나오는 경우
1. 장 시간 확인
2. 채권 종목 코드 정확성 확인
3. 모의투자에서 해당 채권 지원 여부 확인

### 거래 테스트 실패하는 경우
1. 영업일 확인
2. 장 시간 확인 (9:00 ~ 15:30)
3. API 권한 확인

## 주의사항

1. **모의투자 환경 사용**: 실제 거래가 아닌 모의투자로 안전 테스트
2. **영업일 확인**: 모의투자도 영업일에만 작동
3. **API 제한**: 너무 빈번한 호출은 Rate Limit에 걸릴 수 있음
4. **계정 정보**: 실제 계정 번호와 API 키 필요
5. **DB 연결**: PostgreSQL 서버가 실행 중이어야 DB 테스트 가능
6. **실행 환경**: DB 테스트는 실제 DB 연결이 있는 서버에서만 실행
7. **테스트 데이터**: demo 환경에서만 샘플 데이터 생성

## 데이터베이스 테스트 상세

### DB 테스트 기능 (test_database_data)

BitCoin_Auto의 app.py를 참고하여 구현된 데이터베이스 테스트:

1. **연결 확인**
   - DatabaseManager 자동 초기화 및 테이블 생성
   - PostgreSQL 연결 상태 검증
   - 재시도 및 타임아웃 처리

2. **데이터 현황 조회** 
   - trading_history: 거래 기록 (심볼, 주문유형, 수량, 가격)
   - rebalancing_logs: 리밸런싱 실행 로그 (사유, 상태, 주문 건수)  
   - portfolio_snapshots: 포트폴리오 스냅샷 (총자산, 포지션 수)
   - system_logs: 시스템 실행 로그 (로그 레벨, 모듈, 메시지)

3. **샘플 데이터 생성**
   - 테스트용 거래 기록 (삼성전자 1주 매수)
   - 시스템 로그 (테스트 실행 기록)
   - 정상 저장 여부 확인

### 에러 시나리오 처리

- **DB 연결 실패**: 설정 파일 및 PostgreSQL 상태 확인 안내
- **테이블 없음**: 자동 생성 실패 시 초기화 문제 알림  
- **데이터 없음**: 정상 상태로 안내 (초기 설치 시)
- **권한 오류**: PostgreSQL 사용자 권한 확인 요청
