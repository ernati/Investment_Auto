# samsung_auto_trading.py 애플리케이션 문서

## 개요
`samsung_auto_trading.py`는 삼성전자 주식 1주를 시장가로 자동 매수한 후, 2분 대기 후 자동으로 매도하는 애플리케이션입니다.

## 파일 위치
- **경로**: `Investment_Auto/Scripts/apps/samsung_auto_trading.py`
- **유형**: 애플리케이션 (Application)

## 주요 기능

### 프로그램 흐름
1. **설정 로드**: Config/config.json에서 API 인증 정보 로드
2. **API 인증**: 한국투자증권 API 인증 토큰 발급
3. **현재가 조회**: 삼성전자 현재가 및 시장 정보 조회
4. **매수 주문**: 삼성전자 1주 시장가 매수 주문 실행
5. **대기**: 2분(120초) 대기 (10초마다 진행 상황 표시)
6. **매도 주문**: 삼성전자 1주 시장가 매도 주문 실행

### 설정 상수
```python
STOCK_CODE = "005930"      # 삼성전자 종목코드
STOCK_NAME = "삼성전자"     # 종목명
QUANTITY = 1               # 매수/매도 수량
WAIT_SECONDS = 120         # 대기 시간 (2분)
ENV = "real"              # 환경 ('real' 또는 'demo')
```

⚠️ **주의**: 기본값이 'real' (실전 계좌)로 설정되어 있습니다. 테스트 목적으로 'demo' 환경 사용 시 위 값을 'demo'로 변경하세요.

## 사용 모듈

### 1. kis_app_utils
- **기능**: 공통 유틸리티 함수 제공
- **사용**: 
  - `setup_kis_trading_client()`: API 클라이언트 및 거래 클라이언트 설정
  - `print_separator()`: 구분선 출력
  - `print_info()`: 타임스탬프 포함 로그 출력
  - `handle_common_errors`: 에러 처리 데코레이터

### 2. kis_auth
- **기능**: 한국투자증권 API 인증
- **사용**: `setup_kis_trading_client()`에서 내부적으로 사용

### 3. kis_trading
- **기능**: 주식 매수/매도 주문
- **사용**: 
  - `buy_market_order()`: 시장가 매수
  - `sell_market_order()`: 시장가 매도

### 4. kis_api_client
- **기능**: 주식 정보 조회
- **사용**: `get_market_price()`: 현재가 조회

## 상세 기능 설명

### main()
메인 실행 함수입니다. `@handle_common_errors` 데코레이터로 감싸져 있어 자동으로 에러를 처리합니다.

**주요 단계:**
1. 설정 로드 및 인증 (setup_kis_trading_client 사용)
2. 현재가 조회
3. 매수 주문 실행
4. 2분 대기 (진행 상황 표시)
5. 매도 주문 실행

**코드 개선 사항:**
- 공통 유틸리티 함수 사용으로 코드 간소화
- `setup_kis_trading_client()` 함수로 초기화 과정 단순화
- `handle_common_errors` 데코레이터로 에러 처리 자동화
- kis_app_utils 모듈의 `print_info()`, `print_separator()` 사용

## 주의사항

### 1. 환경 설정
- **테스트 시 반드시 `demo` 환경 사용**
  ```python
  ENV = "demo"  # 모의투자 (안전)
  ```
- **실전투자 모드 주의**
  ```python
  ENV = "real"  # 실전투자 (실제 거래 발생!)
  ```

### 2. 시장가 주문 특성
- 시장가 주문은 즉시 체결되지만 **체결가격이 예상과 다를 수 있습니다**
- 급등/급락 시 예상치 못한 가격에 체결될 수 있습니다
- 거래량이 적은 종목은 불리한 가격에 체결될 수 있습니다

### 3. 거래 시간
- **정규장 거래 시간**: 09:00 ~ 15:30 (한국 시간)
- 장 마감 후나 휴장일에는 주문이 실패할 수 있습니다
- 장 시작 전(08:30~09:00) 시간외 거래 가능

### 4. 계좌 잔고
- 매수 시 계좌에 충분한 예수금이 있어야 합니다
- 매도 시 해당 종목의 보유 수량이 있어야 합니다

### 5. 오류 처리
프로그램은 다음과 같은 오류 상황을 처리합니다:
- 설정 파일 없음
- 인증 실패
- API 호출 실패
- 주문 실패

오류 발생 시 콘솔에 오류 메시지가 출력됩니다.

## 에러 해결

### "설정 파일을 찾을 수 없습니다"
```
[ERROR] 설정 파일을 찾을 수 없습니다: Config/config.json
```
**해결**: `Investment_Auto/Config/config.json` 파일이 존재하고 올바른 형식인지 확인

### "매수 주문 실패"
```
[ERROR] ✗ 매수 주문 실패: [40310000] 모의투자 주문처리 불가시간입니다.
```
**해결**: 정규 거래 시간(09:00~15:30)인지 확인

### "인증 실패"
```
[ERROR] 인증 실패: 401, Unauthorized
```
**해결**: Config/config.json의 appkey, appsecret이 올바른지 확인

### 2. 손익률 기반 매도
```python
# 2분 대기 대신 목표 수익률 달성 시 매도
target_profit_rate = 1.0  # 1% 수익
while True:
    current_price = get_current_price()
    profit_rate = (current_price - buy_price) / buy_price * 100
    if profit_rate >= target_profit_rate:
        sell_market_order()
        break
    time.sleep(10)
```

### 3. 스케줄링

```python
import schedule

    time.sleep(1)
```
