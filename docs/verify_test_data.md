# DB 테스트 데이터 검증 스크립트 문서  

## 개요

`verify_test_data.py` 스크립트는 `insert_test_data.py`로 삽입된 테스트 데이터가 데이터베이스에 정상적으로 저장되어 있는지 검증하는 도구입니다. 
웹 인터페이스에서 DB 데이터 시각화를 테스트하기 전에 데이터 무결성을 확인하는 용도로 사용됩니다.

## 주요 기능

### 1. 데이터베이스 연결 검증
- PostgreSQL 데이터베이스 연결 상태 확인
- 테이블 초기화 상태 검증  
- DB 매니저 정상 동작 확인

### 2. 테이블별 데이터 검증
- **거래 기록 (trading_history)**: 매수/매도 거래 데이터 확인
- **리밸런싱 로그 (rebalancing_logs)**: 포트폴리오 리밸런싱 실행 기록 확인
- **포트폴리오 스냅샷 (portfolio_snapshots)**: 시점별 포트폴리오 상태 확인  
- **시스템 로그 (system_logs)**: 시스템 운영 로그 확인

### 3. 데이터 통계 및 요약
- 각 테이블별 데이터 건수
- 최근 데이터 샘플 출력 
- 데이터 유형별 통계 정보

## 검증 함수

### verify_trading_history(db_manager) → bool
거래 기록 데이터를 검증합니다.

**검증 내용:**
- 총 거래 기록 건수
- 최근 5건의 거래 내역 (시간, 종목, 유형, 수량, 가격, 상태)
- 주문 유형별 통계 (매수/매도 건수)
- 주문 상태별 통계 (완료/실패/대기)

**출력 예시:**
```
📈 거래 기록 데이터 확인...
  ✅ 총 15건의 거래 기록 발견
  📊 최근 거래 기록 (최대 5건):
    1. [2026-02-22 23:55:44] KRW deposit 3984259주 @ 1원 - completed
    2. [2026-02-22 23:55:44] 005930 buy 10주 @ 73,414원 - pending
  📋 통계: 매수 6건, 매도 5건, 완료 11건
```

### verify_rebalancing_logs(db_manager) → bool  
리밸런싱 로그 데이터를 검증합니다.

**검증 내용:**
- 총 리밸런싱 실행 건수
- 리밸런싱 사유 및 실행 상태
- 목표 비중 및 실행 결과 
- 실행된 주문 수

**출력 예시:**
```
⚖️ 리밸런싱 로그 데이터 확인...
  ✅ 총 3건의 리밸런싱 로그 발견
  📊 리밸런싱 로그:
    1. [2026-02-22 23:55:44] success - 3건 실행
       사유: 정기 리밸런싱 - 월간 스케줄
       목표 비중: {'005930': 0.5, '035420': 0.3, 'KRW': 0.2}
```

### verify_portfolio_snapshots(db_manager) → bool
포트폴리오 스냅샷 데이터를 검증합니다.

**검증 내용:**
- 총 스냅샷 기록 건수
- 시점별 총 자산 가치
- 포지션 구성 현황
- 스냅샷 생성 시간 분포

**출력 예시:**
```
📸 포트폴리오 스냅샷 데이터 확인...
  ✅ 총 10건의 포트폴리오 스냅샷 발견
  📊 포트폴리오 스냅샷:
    1. [2026-02-22 23:55:45] 총자산: 17,537,157원
       포지션: ['005930', '035420', '000660', 'KRW']
```

### verify_system_logs(db_manager) → bool
시스템 로그 데이터를 검증합니다.

**검증 내용:**  
- 총 시스템 로그 건수
- 최근 로그 메시지 샘플
- 로그 레벨별 통계 (INFO/WARNING/ERROR)
- 모듈별 로그 분포

**출력 예시:**
```
📋 시스템 로그 데이터 확인...
  ✅ 총 21건의 시스템 로그 발견
  📊 최근 시스템 로그 (최대 5건):
    1. [2026-02-22 23:55:55] INFO - web_server
       메시지: 웹 서버 시작
  📋 레벨별 통계: {'INFO': 13, 'ERROR': 4, 'WARNING': 4}
```


## 의존성

### 내부 모듈
- `db_manager.py`: 데이터베이스 연결 및 쿼리 관리
- `database.json`: 데이터베이스 연결 설정

### 외부 라이브러리
- `psycopg2`: PostgreSQL 연결
- `json`: JSON 데이터 파싱

## 설정 요구사항

### 데이터베이스 설정
`Config/database.json`에 올바른 DB 연결 정보 필요:
```json
{
    "database": {
        "host": "localhost",
        "port": 5432,
        "name": "appdb",
        "user": "appuser",
        "password": "temp1234"
    }
}
```

### 테스트 데이터 포트폴리오 ID
테스트 데이터는 `demo_portfolio` ID로 생성되므로, 웹 인터페이스에서도 해당 ID로 조회해야 함.

## 오류 해결

### 1. 데이터베이스 연결 실패
```
❌ 데이터베이스 초기화 실패
```

**해결 방법:**
- PostgreSQL 서버 실행 상태 확인
- `Config/database.json` 설정 점검
- 네트워크 연결 및 방화벽 설정 확인

### 2. 테이블 없음 오류
```
❌ 거래 기록 확인 실패: relation "trading_history" does not exist
```

**해결 방법:**
- DB 테이블이 정상 생성되었는지 확인
- `db_manager.py`의 테이블 생성 로직 점검
- 수동으로 CREATE TABLE 스크립트 실행

### 3. 데이터 없음 경고
```
⚠️ 거래 기록이 없습니다.
```

**해결 방법:**
- `insert_test_data.py` 스크립트 실행
- 포트폴리오 ID 일치 여부 확인 (`demo_portfolio`)
- 환경 설정 확인 (`demo`)
